# smd 1.0.0 Roadmap

This roadmap outlines the development journey and future plans for `smd`, aiming to deliver a robust and user-friendly Markdown conversion tool. My goals span from enhancing conversion capabilities to fostering community contributions, optimizing performance, and ensuring wide accessibility and usability.

## Overview of Features

### Pre-Release (Pre 1.0.0)
- PDF Conversion Enhancements
- Image Handling Improvements
- Initialization and Functionality Improvements

### Post-Release (Post 1.0.0)
- Expansion and Redesign of Themes
- Community Theme Contributions
- Comprehensive Testing Suite
- Performance Enhancements
- Enhancements and New Features

## Detailed Roadmap

### Conversion

#### PDF Conversion Enhancements
- **Objective:** Enhance PDF conversion capabilities to include padding at the start and end of each page for improved readability and aesthetics.

#### Image Handling Improvements
- **Objective:** Implement capturing the entire window content as a single image, including content beyond the visible area by "scrolling".

### Initialization and Functionality Improvements
- **Objective:** Enhance the initialization process and overall functionality, including integration of extracted directory, refinement of functions, introduction of a reset system, optimization of the API communicator, verification checks for consistency, and structured configuration file handling.

### Themes

#### Expansion and Redesign of Themes
- **Objective:** Expand and refine the design of native themes to enhance aesthetics and user experience.

#### Community Theme Contributions
- **Objective:** Enable community contributions of themes to diversify available options and foster community involvement.

### Tests

#### Comprehensive Testing Suite
- **Objective:** Implement a robust testing suite to verify setup, server communication, and error handling align with expectations.

### Execution Time Optimization

#### Performance Enhancements
- **Objective:** Reduce rendering time for Markdown documents, improving efficiency and user satisfaction.

### Enhancements and New Features
- **Objective:** Post-release, introduce enhancements for flexibility, user interface, and functionality, including instance-based `FlavourSettings` functions, depth for help commands, enhanced terminal output, progress counter during initialization, and streamlined writing to `flavours.yaml`.

### Release Goals

#### Distribution and Accessibility
- **Objective:** Enhance distribution and accessibility by releasing on popular package managers and creating a Docker image, promoting widespread adoption and ease of use.

---

This roadmap reflects my dedication to making `smd` an exceptional Markdown conversion tool. By focusing on these strategic areas, we aim to enhance the tool's functionality, usability, and accessibility to meet and exceed the needs of the users.
